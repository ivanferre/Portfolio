The Problem with the Button

Januar 7, 2023

I have reached the seventh page of the tutorial, and the first problem has just happened: the navigation button does not function. Let me explain.

When the screen (or the window, if you resize it on a laptop screen), is not large enough to display the navigation menu properly, the javascript code replaces the navigation bar with a button that looks like three horizontal lines. (UX Designers call it the hamburger icon). Pressing this button should display a vertical navigation menu. This is the feature that does not work as per the tutorial.

Let's be more precise: if you just copy and paste the code from the tutotial (as per January 7th, 2023) the example works fine. But if you follow the instructions to look for the appropriate CDN links at the Bootstrap site, the button doesn't display the navigation bar. The icon changes to acknowledge the button has been pressed, though.

I have tried with a second approach: to use the links for Popper and JS separately, as shown below, and that you may find here.

                <script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.11.6/dist/umd/popper.min.js" integrity="sha384-oBqDVmMz9ATKxIep9tiCxS/Z9fNfEXiDAYTujMAeBAsjFuCZSmKbSSUnQlmh/jp3" crossorigin="anonymous"></script>
                <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0-alpha1/dist/js/bootstrap.min.js" integrity="sha384-mQ93GR66B00ZXjt0YO5KlohRA5SY2XofN4zfuZxLkoj1gXtW8ANNCe9d5Y3eG5eD" crossorigin="anonymous"></script>

Unfortunately, the results have been also unsucessful.

My suspicion is that a change in the Bootstrap framework version 5 demands now to do some setup before, or to call the javascript in a different way. Time to research.
