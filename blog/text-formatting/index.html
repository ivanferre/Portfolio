Text Formatting: Multi-column

February 26, 2023

In the links page, I had very long listings. I thought I could optimize screen usage by splitting the text into two separate columns.

I will first explain what I did not: to create a table. This would have been an obvious choice, should I had more information to display about each reference. Or a simple way to do it when there would only be a grid of words. But what I wanted to do was to split the lists into separate columns, not only to present them more clearly, but also to have them classified by some criteria.

And I wanted to keep learning new features, too.

So I have chosen to display the text in more than one column. This also lets me write lists instead of items separately. Bootstrap's CSS have plenty of classes defined to make this easy. The main trick is to have a container to wrap the rows and columns, and to have a clear idea on how to organize things. It's a very flexible and powerful tool that offers much more than I needed for this simple purpose.

The Bootstrap Grid System is very well explained in this page. It's based on flexbox and is fully responsive. Together with the last link to Mozilla documentation, it also refers to this CSS Tricks flexbox Guide that will explain everything you may want to know about flexbox.

The results as of today are not perfect due to a minor issue that I'm not in the mood of correcting right now. The default values in Bootstrap's CSS define a padding for each column. If the left padding is not stripped off, you will notice a very slight difference on the left margin of the page for the text that are single-column, and the ones multi-column. Right now, I prefer to move on to other text formatting topics.
